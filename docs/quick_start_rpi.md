# Быстрый старт с нуля до рабочего ROS 2 образа

## 1. Предварительные требования

Перед началом убедитесь, что ваша система соответствует следующим требованиям:

- **Операционная система**: Raspberry Pi OS (Debian Bookworm) Lite 64-bit  
- **Подключённый полётный контроллер по USB** (для работы QGroundControl)  
- **Подключённый полётный контроллер по UART** (для связи PX4 <=> Raspberry Pi):  
  - TX Raspberry (GPIO14) → RX4 полётного контроллера  
  - RX Raspberry (GPIO15) → TX4 полётного контроллера  
  - GND → GND (обязательно)  
- **Подключённая светодиодная лента** к GPIO 20 Raspberry Pi  
- **Подключённая камера** к разъёму MIPI0 (сенсор OV5647 или IMX219)  

---

## 2. Установка репозитория

### Клонирование с подмодулями

Репозиторий содержит вложенные модули (submodules). Чтобы склонировать их сразу, выполните команду (при необходимости введите учётные данные GitLab):

```bash
cd ~/
sudo apt update
sudo apt install git -y
git clone --recurse-submodules https://git.sverk.io/software-ros2/sverk-ros2.git
```

### Обновление подмодулей

Если вы уже клонировали репозиторий без подмодулей или хотите обновить их до актуальных версий, выполните скрипт (при необходимости введите учётные данные GitLab):

```bash
cd ~/sverk-ros2
bash scripts/reset_repo.sh
```

---

## 3. Подготовка Raspberry Pi

Запустите скрипт, который установит все необходимые зависимости и настроит систему.  

- Если на дроне камера с сенсором **OV5647**:
  ```bash
  sudo bash ~/sverk-ros2/scripts/install_rpi_ov5647.sh
  ```
- Если на дроне камера с сенсором **IMX219**:
  ```bash
  sudo bash ~/sverk-ros2/scripts/install_rpi_imx219.sh
  ```

После выполнения скрипта Raspberry Pi перезапуститcя. Если подключение не прервалось, то перезагрузите вручную:

```bash
sudo reboot
```

После перезагрузки подключитесь по SSH **с указанием порта 2222** (скрипт изменил стандартный порт SSH на 2222). Вы снова окажетесь в системе как пользователь `pi` (вне контейнера):

```bash
ssh -p 2222 pi@<IP_ДРОНА>
```

Пример:

```bash
ssh -p 2222 pi@192.168.1.140
```

### Добавление в группы
Для запуска команд без `sudo` необходимо добавить пользователя во все необходимые группы
```bash
sudo usermod -aG docker $USER
sudo usermod -aG dialout $USER
sudo usermod -aG tty $USER  
sudo usermod -aG i2c $USER
sudo usermod -aG video $USER
sudo usermod -aG gpio $USER
sudo usermod -aG spi $USER
```
И перезапустить Raspberry Pi
```bash
sudo reboot
```

### Запуск Docker‑контейнера

Перейдите в директорию репозитория и запустите контейнер в интерактивном режиме. При первом запуске будут загружены необходимые образы из интернета — это может занять несколько минут.

```bash
cd ~/sverk-ros2
docker compose up --no-start --build
```

### Автоматический запуск контейнера при включении Raspberry Pi

Чтобы контейнер запускался автоматически при старте системы, выполните:

```bash
sudo cp ~/sverk-ros2/scripts/sverk-ros2-docker.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable sverk-ros2-docker.service
sudo systemctl start sverk-ros2-docker.service
```

### Проверка работоспособности

Запустите скрипт проверки состояния всех компонентов:
```bash
bash ~/sverk-ros2/scripts/check.sh
```

Убедитесь, что все проверки пройдены.

## 4. Подключение к контейнеру

Подключиться к запущенному контейнеру можно двумя способами:

### Через Docker (на самой Raspberry Pi)

```bash
docker exec -it sverk_ros2 bash
```

### По SSH с любого устройства в локальной сети

Порт 22 на Raspberry Pi проброшен внутрь контейнера, поэтому подключение по SSH ведёт сразу в контейнер:

```bash
ssh sverk@<IP_ДРОНА>
```

Пароль: `sverk`

---

## 5. Веб‑интерфейсы для управления дроном

После запуска всех ROS-сервисов становятся доступны два веб‑интерфейса (замените `<IP_ДРОНА>` на реальный IP‑адрес вашей Raspberry Pi).

- **Упрощённый интерфейс (dotmdVisualiser)**  
  Доступен по адресу: `http://<IP_ДРОНА>/` (порт 80)  
  Предоставляет базовые вкладки для мониторинга статуса системы, просмотра логов, выполнения типовых команд и отображения телеметрии.

- **Полный интерфейс (sverk-ros2-ui)**  
  Доступен по адресу: `http://<IP_ДРОНА>:5173`  
  Включает расширенные возможности: визуализацию ROS‑топиков, вызов ROS-сервисов, настройку параметров, управление полётным заданием, просмотр видео с камеры и т.д.. Рекомендуется для полной настройки и активного использования дрона.

## 6. Автозапуск launch‑файла при старте контейнера

При запуске контейнера `sverk_ros2` в фоновом режиме выполняется запуск ROS 2 launch всей системы.
Вмешиваться в этот процесс не требуется. Если вам нужно **остановить** запущенный launch‑файл, войдите в контейнер и завершите процесс:

```bash
pkill -f full_system_real.launch.py
```

**Запускается** launch-файл следующей командой:

```bash
ros2 launch launch_system full_system_real.launch.py
```

### Как отключить автоматический запуск

Чтобы **отключить** автозапуск этого launch‑файла, закомментируйте соответствующую строку в файле `/home/sverk/scripts/start.sh` внутри контейнера, а затем перезапустите контейнер.

1. Войдите в контейнер:  
   `docker exec -it sverk_ros2 bash`
2. Отредактируйте файл:  
   `nano /home/sverk/scripts/start.sh`
3. Найдите строку, начинающуюся с `source /opt/ros/humble/...` и добавьте в её начале символ `#`, чтобы она стала комментарием:  
   ```bash
   # source /opt/ros/humble/setup.bash && source /home/sverk/sverk_ws/install/setup.bash && ros2 launch launch_system full_system_real.launch.py &
   ```
4. Сохраните файл (Ctrl+O, Enter) и выйдите из редактора (Ctrl+X).
5. Перезапустите сервис контейнера на хосте (Raspberry Pi):  
   `sudo systemctl restart sverk-ros2-docker.service`
